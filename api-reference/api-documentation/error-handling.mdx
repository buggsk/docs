title: 'Error Handling'
description: ''
---

If an error is encountered, the API will return a JSON object with a status code of either 400 (Bad Request), 401 (Unauthorized), or 500 (Internal Server Error) .
The JSON object will include a detailed error message. For help, contact carrier.integration@orchestro.ai with information.

## Error Object 


<ParamField body="Code" type="string">
Error Code
</ParamField>


<ParamField body="Code" type="string">
Error Description
</ParamField>


### 401 Unauthorized
```json Response
{ 
"errors": [ 
  { 
	“code”: “ERR-1000”, 
	“message”: “Invalid Token.” 
  } 
] 
}
```

### 400 Bad Request
```json Response
{ 
  "errors": [ 
	{ 
  	"code": "ERR-3001", 
  	"message": "Shipment Date is required." 
	}, 
	{ 
 	"code": "ERR-3003", 
 	"message": "Total number of parcels should be greater than 0." 
	} 
  ] 
} 

```

### 500 Internal Server Error
```json Response
{ 
  "errors": [ 
	{ 
  	"code": "ERR-2025", 
  	"message": "We are having some issue, please resubmit the request." 
	}, 
  ] 
} 
 

```
